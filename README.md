# Flux Atelier Store

Продающий фронтенд многостраничного магазина одежды, разработанный как портфолио-проект под реальные бизнес-задачи e-commerce.

## Что решает этот проект:

### Задача
- Увеличить конверсию из просмотра каталога в добавление товара в корзину и оформление заказа.
- Сократить путь пользователя от первого экрана до checkout без лишних шагов.
- Повысить доверие к магазину за счет понятной структуры, карточки товара с фото/размерами/отзывами и прозрачного сценария покупки.
- Обеспечить комфортный UX на мобильных устройствах, где приходит основная доля трафика.

### Решение
- Реализована MPA-структура интернет-магазина с ключевыми страницами:
  - главная,
  - каталог,
  - страницы категорий,
  - карточка товара,
  - checkout,
  - контакты / о бренде / 404.
- Построен понятный пользовательский путь покупки:
  - Hero с сильным оффером и CTA,
  - переход в каталог,
  - фильтрация/сортировка/пагинация,
  - карточка товара с галереей и выбором размера,
  - корзина и оформление заказа.
- Добавлены UX-механики, повышающие вовлечение и удобство:
  - light/dark theme,
  - reveal-анимации блоков при скролле,
  - sticky mobile header,
  - toast-уведомление о добавлении товара в корзину,
  - адаптивные интерфейсы корзины и форм.
- Реализована клиентская корзина на `localStorage`, что сохраняет товары между страницами и сессиями пользователя.
- Фронтенд подготовлен к работе как автономно (fallback-данные), так и с внешним backend API.

### Результат (по текущей реализации)
- Производительность фронтенда:
  - production build за ~1.9s,
  - итоговый JS-бандл: ~54.57 kB (15.54 kB gzip),
  - итоговый CSS-бандл: ~49.43 kB (8.15 kB gzip).
- Полноценная MPA-структура для SEO и рекламного трафика:
  - `index.html`, `catalog.html`, `women.html`, `men.html`, `outerwear.html`, `accessories.html`, `product.html`, `checkout.html`, `about.html`, `contacts.html`, `404.html`.
- Подготовлена устойчивая база для дальнейшего роста проекта:
  - подключение backend/CRM,
  - расширение каталога,
  - запуск performance-маркетинга,
  - A/B-тестирование коммерческих блоков.

## Почему такой подход выгоден бизнесу

- **Быстрый запуск продаж:** уже есть готовый путь покупки от оффера до checkout.
- **Выше конверсия в заказ:** каталог, карточка товара и корзина спроектированы как единая воронка, а не набор отдельных страниц.
- **Качественный mobile UX:** адаптив, sticky header и быстрые сценарии выбора/покупки уменьшают потери мобильного трафика.
- **Прозрачная масштабируемость:** легко добавлять новые категории, товары, промо-механики и интеграции без пересборки архитектуры.
- **Ниже стоимость развития:** компонентная структура и BEM-подход ускоряют доработки и снижают риск регрессий.
- **Готовность к маркетингу:** проект удобен для SEO-страниц, рекламных кампаний и посадочных сценариев под конкретные категории товаров.

## Техническая часть

### Стек
- HTML5 (MPA)
- Vanilla JavaScript (ES Modules)
- SCSS
- Vite 4

### Подходы разработки
- Компонентная структура: `components / layouts / sections / modules / pages`.
- BEM-нейминг для предсказуемой и поддерживаемой стилизации.
- Адаптив + fluid-подход:
  - `fluid()`
  - `fluid-to-laptop()`
- UX/доступность:
  - `aria-label`, семантическая разметка,
  - `prefers-reduced-motion`,
  - адаптация интерфейса под светлую и тёмную темы.

### Архитектура
- Точка входа: `src/main.js`
- Стили: `src/appStyles.js` + `src/styles/*`
- Ключевые модули поведения:
  - `ThemeSwitcher`
  - `MobileMenu`
  - `Cart`
  - `CatalogFilters`
  - `ProductGallery`
  - `CheckoutForm`
  - `ScrollReveal`
